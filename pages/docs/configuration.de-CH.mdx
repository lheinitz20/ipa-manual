# Konfiguration

Standardmässig wird beim Starten des Frameworks der Port `3000` verwendet. Läuft per Zufall bereits ein anderer Web-Dienst mit diesem Port wird der Port des Next.js Frameworks im Entwicklungsmodus automatisch angepasst.
<Callout type="warning" emoji="⚠️">
Anders als im Entwicklungsmodus kann das Framework in so einem Fall nicht gestartet werden.
</Callout>

<Callout>
Mit dem Befehl `npm run dev -- -p 3005` kann der Port manuell geändert werden.
</Callout>

## Ordnerstruktur

Die Dateien im Stammverzeichnis haben alle ihre Richtigkeit. Die Namen sollten nicht verändert werden.
Das <strong>public-Verzeichnis</strong> hat eine bestimmte Struktur.

![public-files](https://raw.githubusercontent.com/lheinitz20/ipa-manual/main/public/public-files.png)

- Bei den Bildern, welche direkt im <strong>public-Verzeichnis</strong> liegen, handelt es sich um Hintergründe, Logos oder sonstige Bilder welche mit einer Kamera gemacht wurden.
- Im <strong>graphics-Verzeichnis</strong> sind sämtliche selbsterstellte Vektorgrafiken zu finden.
- Die heruntergeladenen Icons befinden sich im <strong>icons-Verzeichnis</strong>.

Um Grafiken/Icons/Hintergründe zu ersetzen, kann die neue Datei gleicht benannt werden wie die bereits Vorhandene und dann direkt in das entsprechende Verzeichnis gezogen werden.

<Callout type="warning" emoji="⚠️">
  Handelt es sich allerdings um ein komplett neues Objekt mit anderem Namen, wird es etwas komplizierter. Somit kommen wir zu den [Anpassungen](/docs/adjustments)
</Callout>
